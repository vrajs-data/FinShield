# 🛡️ FinShield – Digital Payment Fraud Analytics

## 📌 Overview

FinShield is a fraud detection analytics project that analyzes transaction data to identify suspicious patterns and reduce financial risks.

## 🎯 Business Objectives

* Identify high-risk transactions
* Detect unusual spending patterns
* Analyze user behavior
* Build fraud risk scoring
* Reduce financial losses

## 📊 Dataset Features

* Transaction ID
* User ID
* Transaction Amount
* Payment Method
* Location
* Risk Score
* Fraud Label
* Transaction Time

## 🛠️ Tech Stack

* Python (Pandas, NumPy)
* MySQL
* Power BI
* DAX

## 📐 Methodology

1. Data Cleaning & Preprocessing
2. Feature Engineering
3. Risk Score Calculation
4. Dashboard Creation
5. Insight Generation

## 📊 Dashboard Features

* KPI Metrics (Transactions, Fraud Rate, Avg Value)
* Time Analysis
* Location-wise Fraud Analysis
* Payment Method Analysis
* Risk Level Distribution

## 🔍 Key Insights

* Higher fraud in specific locations (e.g., UK)
* UPI & Net Banking show higher fraud rates
* Fraud peaks at specific hours
* High-risk transactions contribute most fraud

## 💼 Business Impact

Helps businesses detect fraud early, reduce losses, and improve transaction security.

## 📂 Project Files

* Notebook: Jupyter analysis
* SQL: Database & risk scoring logic
* Data: Transaction dataset
* Docs: Project report
